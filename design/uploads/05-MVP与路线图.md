# 同舟 · MVP 范围与开发路线图（v0.1）

> MVP 目标：让 **1 位创作者** 在同舟上完整跑完 **1 期 21 天训练营**，并产生可量化的"留存 / 完课 / 复购"数据。

---

> **v0.3 关键调整（加速起步）**：把原 8 周 MVP 进一步分层为 V0 → V0.5 → V1 → V2 渐进路线。
> - **V0（4 周）= 轻量化课程交付 + Freemium + 按量计费**——本文档下文称 "V0 范围"，详见 07 文档
> - **V0.5（+2 周）= 微信增强**：公众号订阅消息 + 第三方付费 Webhook
> - **V1（+4 周）= 成长系统补全**：作业 + 签到 + 班级 + AI 反馈
> - **V2（+4 周）= 群运营 + AI 群日报**
>
> 「MVP」这一称呼现在专指 V0；之前 v0.2 路线图里 8 周的内容现在拆为 V0 + V0.5 + 部分 V1。

---

## 1. MVP 定义

### 1.1 V0 的"完成"长这样

> 张老师注册同舟 → 30 分钟内上传 1 门视频课程 → 拿到 1 个分享链接 → 发到自己的微信群 / 朋友圈 → 学员点开 → 看到张老师品牌的 H5 → 选择课时 → 看视频。
>
> 张老师在后台能看到：
> - 谁看了，看了多少
> - 当月学员数 / 视频时长 / 存储 用了多少
> - 距离 Freemium 上限还有多少
> - 一键升级付费档（占位）

**V0 不做：作业、签到、班级、社群、AI、自定义域、微信小程序、微信支付。**
（详见 07 文档 §7）

### 1.2 V0 **要做**

| 模块 | 范围 |
|---|---|
| 多租户 SaaS | Tenant 注册、品牌（Logo / 主色 / 子域名）|
| 课程结构 | Track（课程）+ Session（课时）两层，Stage 层 schema 保留但 UI 隐藏 |
| 内容 | 视频 OSS 直传 + VOD 转码 + 防盗链 + 学员水印 |
| 学员名单 | 单加 + CSV 导入 + 邀请短链（V0.5 加 Webhook）|
| 学员端 H5 | 极简：分享链接落地页 + 课程目录 + 视频播放（不做 PWA、不做 4 Tab）|
| 学员身份 | 匿名 token 默认；可选手机号绑定记录跨设备进度 |
| **用量计量** | 4 维度（学员 / 课程 / 存储 / 播放时长）+ 月对账 + 重算 |
| **Freemium quota** | 字段预留 + 创作者后台 quota 进度条 + 超额降级路径 |
| 看板 | 创作者后台：每个课程谁看了多少 + 用量进度 |
| 通知 | 邮件 + 站内（V0 不做公众号 / 短信）|

### 1.3 V0 **不做**

| 模块 | 后置到 | 后置原因 |
|---|---|---|
| 作业 / 签到 / 班级 | V1 | V0 验证课程交付即可 |
| 成长路径 / Stage | V1 | 数据层保留，UI 不做 |
| 群运营助手 / AI 反馈 | V1+ | 不是冷启动必需 |
| 公众号订阅消息 | V0.5 | V0 用邮件 + 站内通知 |
| 第三方付费 Webhook | V0.5 | V0 仅 CSV / 单加 |
| 微信 OAuth2 | V0.5 | V0 学员匿名 + 手机号 |
| PWA / 添加桌面 | V0.5 | 极简 H5 即可 |
| 微信小程序 | V2 | 先验证 H5 够不够用 |
| 微信支付 / 分账 | 永不强收 | 钱在创作者自有渠道，同舟不碰资金 |
| 直播 / 回放 | V2 | — |
| 自定义域名 | V1 | — |
| 团队成员 / Coach | V1 | — |
| 学员评论 / 互评 | V1 | — |
| 装修系统 / 商城 / SKU | 永不做 | 与定位冲突 |

## 2. 关键里程碑（v0.3 · V0 4 周 + 渐进）

> 假设：2 人前端 + 2 人后端 + 1 人产品/设计兼项目
> **v0.2 → v0.3 变更：** 把"训练营完整闭环"压缩为"轻量课程交付（V0）"先验证商业可行性，然后逐步加重。

```
─────── V0：轻量课程交付（4 周）───────────
W0   ─ 需求/设计收口 + 基建准备
W1   ─ 基建 + Tenant + 登录 + 多租户骨架
W2   ─ 视频上传/转码 + 课程 + 课时
W3   ─ 学员 H5 + 播放器 + 进度记录
W4   ─ 计量 + Freemium quota + 看板 + 灰度首批 ← V0 Done

─────── V0.5：微信轻增强（+2 周）───────────
W5   ─ 公众号订阅号 + 订阅消息 + OAuth2 + 短信
W6   ─ 第三方付费 Webhook（首期 1 家）+ PWA 添加桌面

─────── V1：成长系统补全（+4 周）───────────
W7   ─ Stage + 作业系统（文字/图/链接）
W8   ─ 签到 + 班级（Cohort fixed_term）
W9   ─ AI 作业反馈初稿 + 内容安全
W10  ─ 团队成员 + Coach + 自定义域

─────── V2：群运营 + AI（+4 周）───────────
W11  ─ 群绑定 + 群文案生成
W12  ─ 群聊天记录粘贴 → AI 精华 / 日报
W13  ─ FAQ 自动生成 + 学员画像
W14  ─ 联调 + 完整训练营首营
```

### 2.1 W0：需求收口

- 评审本套设计文档
- 锁 V0 范围 + 北极星指标 + 验收标准
- **锁 Freemium 阈值（4 个维度）+ 按量计费单价**（你来定，可先内部预估）
- 锁定首批 5 位 friendly creator（V0 阶段）
- 阿里云账号 / VOD / OSS 开通

### 2.2 W1：基建周（"宽地基"）

| 任务 | 产出 |
|---|---|
| 仓库脚手架（monorepo: pnpm + turborepo） | apps/web、apps/api、apps/miniprogram、packages/* |
| 基础设施（K8s / 数据库 / Redis / OSS） | dev / staging 两套就绪 |
| 多租户中间件 + RLS 启用 + 反向测试 | "无 tenant 上下文必拒"的 CI 测试通过 |
| 登录 / 鉴权 | 创作者后台邮箱登录 + 学员小程序 wx.login |
| CI/CD | PR → dev 自动部署 |
| 可观测性 | 日志 / Metrics / Trace 接入 |

**出关条件：** 一位新工程师 git clone → 30 分钟内能本地跑通 hello world。

### 2.3 W2：视频上传 + 课程 / 课时

- OSS 直传凭证发放接口
- VOD 转码回调 + 视频状态机
- Track 创建 / 列表 / 详情（隐藏 Stage）
- Session 创建 / 排序 / 视频绑定
- 视频防盗链 playAuth + 水印参数

### 2.4 W3：学员 H5 + 播放器 + 进度

- 学员 H5 极简版（落地页 + 课程目录 + 播放）
- 匿名 token cookie 机制
- 可选手机号短信绑定（按需）
- 播放器（VOD SDK，带水印）+ 30 秒心跳上报
- 进度记录 + 跨设备继续看

### 2.5 W4：计量 + Freemium + 看板 + 灰度

- `usage_events` / `usage_meters` 双层数据流
- 4 维度计量采集到位（学员 / 课程 / 存储 / 播放分钟）
- `tenant_quotas` 后台可改、热生效
- 超额降级路径全跑通（含"学员观看不受影响"红线）
- 创作者后台看板：用量 + 学员观看
- **灰度首批：邀请 5 位 friendly creator 试用**
- V0 Done

### 2.6 W5-W6：V0.5 微信轻增强

W5：
- 同舟自营订阅号申请 + 接入
- 公众号 OAuth2 静默拿 openid + 与匿名 token 合并
- 公众号订阅消息 5 个核心模板 + quota 维护
- 短信兜底通道（阿里云）

W6：
- 第三方付费 Webhook（首期 1 家：小报童 或 知识星球）
- PWA Manifest + Service Worker + 添加到桌面引导
- 邀请短链规范化

### 2.7 W7-W10：V1 成长系统补全

W7：Stage 层 UI 启用 + 作业系统（文字/图/链接，OSS 直传）
W8：签到 + 连签 + 班级（Cohort `fixed_term`）+ 报名排程
W9：AI 作业反馈初稿（DeepSeek + Claude 双模型）+ 内容安全
W10：团队成员 + Coach 权限 + 自定义域 + V1 首营

### 2.8 W11-W14：V2 群运营 + AI

- 群绑定 + 群文案生成
- 群聊天记录粘贴 → AI 精华 / 日报
- FAQ 自动生成 + 学员画像
- 完整训练营 21 天灰度

## 3. V0 验收标准（DoD）

### 3.1 功能 DoD

- [ ] 创作者 30 分钟内：注册 → 上传 1 段视频 → 发布 1 门课 → 拿到分享链接
- [ ] 学员 1 分钟内：点链接 → 看到课程目录 → 播放视频
- [ ] 计量四维度实时累计、月对账可重算
- [ ] Freemium 字段可改、可热生效
- [ ] 超额降级路径全跑通（**学员观看不受影响**红线）

### 3.2 质量 DoD

- [ ] 关键接口 P95 < 03 §8 性能预算
- [ ] 多租户跨读零次（自动化测试 + 人工渗透各一轮）
- [ ] 视频盗链测试通过
- [ ] 计量数据可重算（重跑事件流得到一致结果）

### 3.3 业务 DoD

- [ ] 招募 5 位 friendly creator 完成首月使用
- [ ] 至少 3 位创作者上传 ≥ 1 门课并产生学员观看
- [ ] 至少 1 位创作者主动咨询升级付费

## 4. 风险与应急预案

| # | 风险 | 缓解 |
|---|---|---|
| M1 | 第三方付费工具 Webhook 不稳定或断更 | CSV 周期导入兜底；首期适配 1 家，验证再扩 |
| M2 | 视频转码不稳定 | provider 抽象层 + 双云备份（VOD A/B） |
| M3 | 单期学员量不足 30 | 与 friendly creator 提前对齐，必要时同舟提供推广支持 |
| M4 | AI 反馈质量差 | 准备 prompt 库 + 创作者反馈循环 + 切模型 fallback |
| M5 | 微信浏览器对 H5 限制收紧 | PWA + 多触达通道；保底切外部浏览器场景 |
| M6 | 团队协作效率 | 周一规划 / 周五演示节奏；ADR 文化制度化 |

## 5. V1 / V2 蓝图（路线图鸟瞰）

### V1（MVP 后 4-8 周）

- 直播接入（腾讯云 / 视频号小程序直播）
- 团队成员 + Coach 权限
- 自定义域名（CNAME + 证书自动签发）
- **加重路径**：可选接入微信支付（服务商分账），让一部分付费用户走完全自营链路
- 作业互评 + 徽章 + 排行
- iOS Web Push 启用

### V2（MVP 后 8-16 周）

- 微信小程序（**真正的"加重"决策点**：仅当 H5 体验明显瓶颈或大客户强诉求时启动）
- 企业微信外部群 + 群机器人（可读群消息）
- 群消息真正结构化（自动精华 / FAQ / 学员画像）
- 多语言（英文版本，海外华人市场）
- API 开放（创作者可接自有系统）

### V3（远期）

- AI 学习助教（端到端答疑）
- 创作者增长引擎（裂变 / 复购自动化）
- 同舟应用市场（创作者交付模板 / 增长模板）

## 6. 团队节奏

- 周一上午：规划周会，回顾上周指标 + 本周计划
- 每日：15 分钟站会
- 周五下午：Demo + 复盘
- 隔周：ADR 评审 1 小时
- 月度：用户访谈 ≥ 5 位创作者

## 1. 端到端形态总览（v0.2 · C 路线）

同舟整体有三个"前台"：

```
┌────────────────────────────────────────────────────────────────────┐
│                          同舟平台                                  │
│                                                                    │
│  ┌───────────────────┐  ┌────────────────┐  ┌──────────────────┐  │
│  │ 创作者后台 (Web)  │  │ 学员 H5 + PWA  │  │ 平台运营后台     │  │
│  │  Next.js H5       │  │  Next.js H5    │  │ Next.js H5       │  │
│  │  app.tongzhou.app │  │  m.tongzhou.app│  │ admin.tongzhou   │  │
│  │                   │  │ 或 xxx.com/m   │  │                  │  │
│  └───────────────────┘  └────────────────┘  └──────────────────┘  │
│           ▲                      ▲                  ▲             │
│           │                      │                  │             │
│           └──────────────────────┼──────────────────┘             │
│                                  ▼                                │
```

**辅助触点：**

- 公众号（订阅号即可，订阅消息推送 + 底部菜单跳 H5）

## 3. 学员 H5 / PWA 信息架构

> 极简优先。一个学员只关心：今天看什么、今天交什么、自己长成什么。
> 载体：H5（Next.js），通过 PWA 提升二次打开体验；微信浏览器内通过 OAuth2 静默拿 openid。

### 3.0 V0 阶段：极简单页（不是 4 Tab）

> V0 时学员端只有 **3 个页面**：

```
1. 落地页 /m/t/:tenantSlug/c/:courseSlug      课程概览
   ┌──────────────────────────────────┐
   │ [创作者 Logo / 头像]              │
   │ 「张老师 · 21 天写作训练营」      │
   │                                   │
   │ 简介 ...                          │
   │                                   │
   │ ▶ 课时 1：第一步                  │
   │ ▶ 课时 2：第二步                  │
   │ ▶ 课时 3：（未开放）              │
   │                                   │
   │ [加群联系老师] (创作者预设链接)    │
   └──────────────────────────────────┘

2. 播放页 /m/t/:tenantSlug/c/:courseSlug/l/:lessonSlug
   ┌──────────────────────────────────┐
   │ [视频播放器，带学员水印]          │
   │                                   │
   │ 课时标题                          │
   │ 课时简介                          │
   │                                   │
   │ ← 上一课时   下一课时 →           │
   └──────────────────────────────────┘

3. 「我的」抽屉（可选绑定手机号）
   ┌──────────────────────────────────┐
   │ 当前为访客模式（仅本设备）         │
   │ [绑定手机号 跨设备同步]            │
   │                                   │
   │ 我看过的课程：                    │
   │ · 张老师 · 21 天写作训练营 (60%)  │
   └──────────────────────────────────┘
```

**V0 学员端不做：** 4 Tab、班级动态、签到、作业、PWA。
**V0.5 起加：** 接微信 OAuth2、PWA 添加到桌面、订阅消息授权。
**V1 起加：** 4 Tab 完整版（今日 / 路径 / 班级 / 我的）。

## 4. 关键用户流程（User Flows）

### 4.1 创作者新建并开营一期训练营

```
[登录后台]
     ▼
[创建 Track] → 命名、简介、封面、定价（仅展示用，不在同舟收钱）
     ▼
[设计 Stage / Session 结构]
     ▼
[为每个 Session 上传视频 + 添加作业模板 + 物料]
     ▼
[创建 Cohort（一期）] → 起止时间、容量、Session 排程
     ▼
[绑定微信群（生成绑定码）] → 创作者把绑定码发到群里 / 后台输入群昵称
     ▼
[导入学员名单] →
   ├ 单个添加（手机号 + 昵称）
   ├ CSV 批量导入
   └ Webhook 接第三方（V1 接知识星球/小报童）
     ▼
[生成报名链接] → 短链 / 海报 / 二维码
     ▼
[开营] → 系统按排程开放 Session、推送公众号订阅消息 + 群文案
```

### 4.2 学员日常学习闭环

```
[公众号订阅消息 / 群通知: "今日 Session 已开放"]
     ▼
[点开链接 → H5「今日」页]
     ▼
[看视频] (产生 progress)
     ▼
[领取今日任务] → 签到 + 作业
     ▼
[提交作业（文字/图/链接）]    (MVP 不接音视频上传)
     ▼
[去班级动态看同学 / 点赞]
     ▼
[领取打卡徽章 / 看到自己的连签 +1]
     ▼
[次日: 订阅消息 / 群文案再次拉回]
```

### 4.3 创作者批阅作业（一天 5 分钟版）

```
[后台 → 班级 → 作业]
     ▼
[今日待批阅列表（按 cohort 过滤）]
     ▼
[逐条查看 → AI 给出反馈初稿]
     ▼
[修改两句 → 发送] → 推送到学员小程序 + 公众号 + 可选同步群里
     ▼
[标记精华 → 进入班级动态置顶]
```

### 4.4 创作者社群运营（群智能助手）

```
[后台 → 社群中心 → 群运营助手]
     ▼
[选择 cohort / 群]
     ▼
A. 生成今日开营文案 → 一键复制 → 创作者粘贴到群
B. 粘贴近 N 条聊天记录 → AI 生成精华 / 日报 → 复制发回群
C. 查看本群学习数据 → 谁未签到 / 谁掉队 → 生成 @ 名单文案
```

### 4.5 学员报名进群链路（C 路线核心入口）

```
[学员通过创作者的现有渠道付费]
   ├ 知识星球：付费入星
   ├ 小报童 / 有赞：付费表单
   └ 私下：微信收款 / 红包
     ▼
[创作者拿到付费名单后 → 同舟后台导入或 Webhook 同步]
     ▼
[同舟生成专属入营短链 → 通过群 / 私聊发给学员]
     ▼
[学员点开 → H5 落地页]
     ▼
[手机号短信验证 / 微信内 OAuth2 静默拿 openid]
     ▼
[绑定到 cohort 中已预录的 member 记录]
     ▼
[引导：开启提醒 + 加群 + 添加到桌面]
     ▼
[进入「今日」首页]
```

## 5. 状态机：Track / Cohort / Member / Session 进度

### 5.1 Cohort 生命周期

```
draft → enrolling → in_progress → archived
                    ↑
                    paused（可选）
```

### 5.2 Session（学员视角）的可见状态

```
locked      → 未开放（按排程未到）
available   → 已开放未学习
in_progress → 视频已看部分
completed   → 视频看完 + 作业已交
overdue     → 已过截止日未完成
```

### 5.3 Submission（作业）状态

```
draft → submitted → reviewed
           │
           └→ revise_requested → submitted（重新提交）
```

## 6. 信息架构里的关键决策

| 决策点 | 选择 | 理由 |
|---|---|---|
| 是否做"课程目录"页 | ❌ 不做 | 与"成长路径优先"原则冲突 |
| 学员是否能跨 Track 浏览 | ❌ 不可 | 学员只看自己加入的 Track |
| 班级动态是否全员可见 | ✅ 是 | 推动同伴学习 |
| 学员是否能点评同学作业 | 🟡 V2 | MVP 仅点赞 |
| 是否做"全平台广场" | ❌ 不做 | 避免内容污染 + 数据隔离 |
| 后台是否做移动端 | 🟡 V2 | 创作者主力在 PC，移动端做"轻批阅"即可 |

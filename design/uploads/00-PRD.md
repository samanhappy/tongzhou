## 0. V0 卷首（必读）

**长期定位** = 面向创作者的微信成长社群 SaaS 操作系统（见 §1）
**V0 切入** = 轻量化课程交付 SaaS

我们采用「加速起步」路径：V0 以"比小鹅通更轻、更便宜"的课程交付能力冷启动，吸引付费创作者；V0.5 加微信增强；V1 加成长系统（作业 / 签到 / 班级 / AI）；V2 加群运营。详见 [07-V0范围与计量模型.md](./07-V0范围与计量模型.md) 与 [05-MVP与路线图.md](./05-MVP与路线图.md)。

- 架构层完整保留长期愿景的所有领域模型与模块边界
- V0 实现层是 V1 的真子集，V1 是 V2 的真子集

## 1. 产品定位

**一句话定义：**
> 同舟，是面向创作者的微信成长社群 SaaS 操作系统。

**它不是什么（强约束）：**

- 不是知识付费课程平台（小鹅通 / 千聊 模式）
- 不是 LMS（在线教育系统）
- 不是 IM 工具（不替代微信）
- 不是电商 / 商城系统
- 不是单点功能工具

**它是什么：**

- 让创作者像运营一个 SaaS 产品一样，运营自己的成长社群与训练营体系
- 三层产品结构：SaaS 平台层 + 成长社群交付层 + 微信生态增强层
- 每位创作者拥有独立的 Tenant 空间，数据完全隔离

## 2. 用户画像

### 2.1 主用户：创作者（Tenant Owner）

| 维度 | 描述 |
|---|---|
| 身份 | 知识 IP、教练、咨询师、自由职业讲师、小型培训机构主理人 |
| 现状 | 在微信群里跑训练营，靠 Excel / 飞书表单 / 群公告 + 人工管理 |
| 规模 | 单期学员 30 – 500 人，团队 1 – 5 人 |
| 痛点 | 群消息淹没、作业难追踪、留存难量化、复购难驱动、品牌不专业 |
| 付费动机 | 提升交付体验 → 提升留存与复购 → 降低运营成本 |

### 2.2 次用户：学员（Member）

| 维度 | 描述 |
|---|---|
| 身份 | 普通微信用户，在某创作者的训练营中学习 |
| 设备 | 微信内为主，偶尔进 H5 / 小程序 |
| 行为习惯 | 不愿装 App，习惯在群里被推送提醒 |
| 期望 | 看视频、交作业、打卡、被回应、看到自己的进步 |

### 2.3 协作者：助教 / 班主任（Coach，可选）

| 维度 | 描述 |
|---|---|
| 身份 | 创作者团队成员，负责答疑、批作业、群运营 |
| 权限 | 受限于创作者授权，只能操作授权的训练营 |

### 2.4 平台方：同舟运营（Platform Admin）

负责 Tenant 审核、订阅计费、平台数据、产品迭代。

## 3. 核心使用场景（Top 5）

### 场景 1：创作者开一期新训练营

> 张老师要开"21 天写作训练营"。她在同舟后台配置训练营信息、上传视频、设置每日 Session。学员通过她现有的小报童 / 知识星球付费 → 同舟通过 Webhook 或 CSV 拿到名单 → 系统自动生成入营 H5 短链 → 张老师把链接发到群里。

### 场景 2：学员每日学习与作业

> 学员小李在微信群收到"今日 Session 已开放"的卡片或公众号订阅消息。点开 → 进入创作者品牌的 H5 站 → 看 5 分钟引导视频 → 提交今日作业（文字 / 图 / 链接） → 看到同伴作业 → 领取打卡徽章 → 系统生成"未签到名单"文案，张老师粘贴到群里 @ 提醒。

### 场景 3：创作者运营群与回应

> 张老师打开同舟后台 → 看到今日"群智能日报"：12 条作业、3 条提问、2 条精华、5 位未签到学员。她直接在后台批量点评作业，AI 给出回复初稿，她改两句发送，系统自动同步到群里。

### 场景 4：训练营结营 → 复盘 → 复购

> 21 天结束。同舟自动生成"班级成长报告"（完课率、作业完成率、优秀学员榜）+ 每位学员的"个人成长档案"。张老师一键开下一期，已有学员复购优惠自动生效。

### 场景 5：创作者升级品牌

> 张老师替换 Logo、主色、域名（`m.tongzhou.app/t/zhang` 或自定义域 `class.zhangsays.com/m`），整个学员 H5 立刻变成她的品牌，对外不见"同舟"字样。

## 6. 非目标（再次强化）

系统**不做**：

- 重型 LMS（章节、考试、证书体系）
- 电商商城（SKU、库存、物流）
- 独立 IM（不替代微信）
- PDF 知识库 / 文档库（不做飞书 / Notion）
- CRM（不做销售管道）
- 短视频社区（不做小红书 / 抖音）

**取舍原则：** 凡是让创作者「离开微信」的功能，原则上都不做。

---

## 7. 关键产品原则（指导决策）

1. **SaaS 优先**：每个功能必须支持 tenant_id 维度的隔离与配置。
2. **WeChat-first**：能在微信里完成的就不要跳出微信。
3. **成长优先，而非内容优先**：不做"课程列表"，做"成长路径"。
4. **视频 = 触发器，社群 = 驱动器，作业 = 闭环，签到 = 习惯**。
5. **AI 是助手，不是发声者**：AI 出初稿，人决定发不发。
6. **轻量化于小鹅通**：少功能，但每个功能都对成长目标负责。

# 同舟 · V0 范围与计量模型（v0.1）

> V0 定位：「**加速起步版**」——以「轻量化课程交付」为切入点冷启动，架构保留完整成长路径骨架，不堵 V1/V2 路。
> 设计原则：**同舟只交付，不收钱**；多维度计量预留；Freemium 字段预留具体阈值后定。

---

## 1. V0 一句话定位

> **一个比小鹅通更轻、更便宜、更适合个体创作者的"知识视频交付 SaaS"。**

| 维度 | 小鹅通 | 同舟 V0 |
|---|---|---|
| 起步价格 | 4800/年 | **Freemium 免费起步** |
| 计费模式 | 订阅 + 抽成 | **按用量计费**（综合维度） |
| 创作者门槛 | 需小程序 + 公众号 + 商户号 | **0 资质要求** |
| 售卖能力 | 强（商城、SKU、营销） | **不卖货**，创作者用自己的渠道 |
| 学员端 | 小程序 + H5 | **H5 落地页** |
| 课程结构 | 章节 + 课时 | **课程 + 课时**（极简） |
| 装修系统 | 重型 | **无装修，模板化** |

**取舍核心：** 砍掉"商城 + 装修 + 营销 + 多种售卖玩法"，只留"上传视频 → 学员看视频"主链路；用 Freemium + 按量定价吸引"刚起步的小创作者"。

```
                                                   V2: 群运营 + AI 群日报 +
                                                       FAQ 自动化
                                                          ▲
                                                          │
                                                V1: 作业 + 签到 + 班级 +

                                                          ▲
                                                          │
                                              V0.5: 报名链路 +

                                                          │
                                     V0: 轻量化课程交付  ← 当前
```

**架构保留原则：**
- 数据模型完整保留 Track / Stage / Session / Cohort / Member（V0 不创建 Cohort 实例，仅用默认 rolling cohort）
- 模块结构按 03 文档完整划分，V0 阶段未使用的模块留空
- 任何 V0 的"简化"都是**实现层简化**，**不是模型层简化**

## 4. 多维度计量模型（核心新增）

> 既然按"综合用量"计费，必须**实时累计 + 可对账 + 可降级**。

### 4.1 计量维度（4 个）

| Metric Key | 单位 | 累计方式 | 采集点 |
|---|---|---|---|
| `members.active_count` | 人 | **当月去重** | 学员任意访问视为活跃 |
| `courses.count` | 门 | **当月最大值** | 课程 status=`published` 的总数 |
| `storage.bytes` | GB·月 | **每日快照求平均** | OSS bucket 用量定时统计 |
| `playback.minutes` | 分钟 | **当月累加** | 播放器心跳上报 |

### 4.2 计量数据流

```
                    ┌────────────────────────────┐
                    │ 业务事件 / 定时快照         │
                    │  - 学员访问                 │
                    │  - 视频心跳                 │
                    │  - 课程发布                 │
                    │  - OSS 日用量同步           │
                    └────────────┬───────────────┘
                                 │
                                 ▼
                    ┌────────────────────────────┐
                    │ usage_events (原子事件流)   │
                    │  tenant_id, metric_key,     │
                    │  delta, occurred_at, ref    │
                    └────────────┬───────────────┘
                                 │
                  (按 tenant + metric + month 滚动累加)
                                 │
                                 ▼
                    ┌────────────────────────────┐
                    │ usage_meters (月聚合)       │
                    │  tenant_id, metric_key,     │
                    │  period (YYYY-MM),          │
                    │  current_value,             │
                    │  free_quota,                │
                    │  billable_value (=max(0,    │
                    │     current - quota))       │
                    └────────────┬───────────────┘
                                 │
                  (每月 1 日生成 invoice / 用量账单)
                                 ▼
                    ┌────────────────────────────┐
                    │ usage_invoices              │
                    │  status: draft/issued/paid  │
                    └────────────────────────────┘
```

### 4.3 表设计

#### `usage_events`（事件流，原子级）

| 字段 | 类型 | 说明 |
|---|---|---|
| tenant_id | uuid NOT NULL | |
| id | uuid PK | |
| metric_key | text | `members.active_count` / `courses.count` / `storage.bytes` / `playback.minutes` |
| delta | numeric | 增量（正负皆可） |
| ref_kind | text NULLABLE | `member` / `video` / `course` / `snapshot` |
| ref_id | uuid NULLABLE | |
| occurred_at | timestamptz | |
| meta | jsonb | 额外信息（如 IP、设备） |

> 索引：`(tenant_id, metric_key, occurred_at)`

#### `usage_meters`（月聚合，幂等可重算）

| 字段 | 类型 | 说明 |
|---|---|---|
| tenant_id | uuid NOT NULL | |
| id | uuid PK | |
| metric_key | text | |
| period | text | `YYYY-MM` |
| current_value | numeric | 当前累计（每日异步刷新） |
| free_quota | numeric | 该 Tenant 当月免费额度（从 `tenant_quotas` 取） |
| billable_value | numeric GENERATED | `GREATEST(0, current_value - free_quota)` |
| unit_price | numeric NULLABLE | 计费单价快照，每月生成账单时定格 |
| updated_at | timestamptz | |

> 唯一：`(tenant_id, metric_key, period)`

#### `tenant_quotas`（每个 Tenant 的 Freemium 阈值，字段预留）

| 字段 | 类型 | 说明 |
|---|---|---|
| tenant_id | uuid PK | |
| plan | text | `free` / `pro` / `custom` |
| effective_at | timestamptz | 生效时间（升降级时分段） |
| quotas | jsonb | 见下方结构 |

**`quotas` 结构（预留，具体数值后定）：**

```json
{
  "members.active_count":  { "free": null, "unit_price": null, "currency": "CNY" },
  "courses.count":         { "free": null, "unit_price": null, "currency": "CNY" },
  "storage.bytes":         { "free": null, "unit_price": null, "currency": "CNY", "unit": "GB·月" },
  "playback.minutes":      { "free": null, "unit_price": null, "currency": "CNY", "unit": "千分钟" }
}
```

> `free` 与 `unit_price` 暂留 `null`，由运营在内部后台手工填写，验证一个月后再固化为代码常量。

#### `usage_invoices`（账单）

| 字段 | 类型 | 说明 |
|---|---|---|
| tenant_id | uuid NOT NULL | |
| id | uuid PK | |
| period | text | `YYYY-MM` |
| status | text | `draft` / `issued` / `paid` / `void` |
| total_amount | numeric | |
| lines | jsonb | 每个 metric 一行：metric_key, billable, unit_price, subtotal |
| issued_at | timestamptz NULLABLE | |
| paid_at | timestamptz NULLABLE | |

## 5. Freemium 与降级策略

> 阈值由你后定。这里只规定"超额发生时怎么处理"，规避"突然停服"的体验灾难。

### 5.1 三档信号

| 用量比例 | 状态 | 行为 |
|---|---|---|
| 0 – 80% | `healthy` | 静默 |
| 80% – 100% | `warning` | 创作者后台横幅 + 邮件提示 |
| 100% – 120% | `over_quota_soft` | 横幅升级为红色 + 上传新视频按钮置灰；**已发布内容正常播放** |
| > 120% | `over_quota_hard` | 仅 7 天宽限：宽限期内创作者继续付费访问 + 学员正常看；7 天后**仅创作者侧停写**，学员侧仍可继续看，直到付费或主动注销 |

**红线：学员观看体验不能因为创作者超额被打断。** 这是与小鹅通的差异化承诺。

### 5.2 降级路径表

| 维度 | 软超额 | 硬超额 7 天后 |
|---|---|---|
| 上传视频 | ❌ | ❌ |
| 添加学员 | ❌ | ❌ |
| 发布新课程 | ❌ | ❌ |
| **学员看视频** | ✅ | ✅ |
| 学员手机号绑定 | ✅ | ✅ |
| 创作者编辑已有元数据 | ✅ | ❌ |
| 创作者后台只读 | ✅ | ✅ |

## 6. 关键架构决策（保留性 vs 简化性）

| 决策 | V0 实现 | 保留原因 |
|---|---|---|
| Track / Stage / Session 三层 | V0 隐藏 Stage 层，UI 只展示 Course / Lesson | V1 加成长路径时需要 |
| Cohort 班级 | V0 每个 Track 自动创建一个 `default rolling cohort` | V1 加期次班级时无需重构 |
| Member ↔ Track 关系 | 通过 Enrollment 关联到 default cohort | 保持外键统一 |
| 业务表 tenant_id + RLS | **完整保留** | 这是产品魂 |
| 模块化单体 | **完整保留** | V1 加功能就是开新模块 |
| API 命名空间 | V0 接口是 V1 的子集 | V1 直接 ADD，不破坏 |

> **核心心法：** V0 是 V1 的真子集，V1 是 V2 的真子集。任何"V0 临时方案"都必须可被 V1 自然替换，不能埋技术债。

## 8. V0 验收 DoD

### 8.1 功能 DoD

- [ ] 创作者 30 分钟内：注册 → 上传 1 个视频 → 发布 1 门课 → 拿到分享链接
- [ ] 学员 1 分钟内：点开链接 → 看到课程目录 → 播放视频（含水印）
- [ ] 计量四维度实时累计、月对账可重算
- [ ] Freemium 字段在后台可改、可热生效
- [ ] 超额降级路径全跑通（含"学员观看不受影响"红线）

### 8.2 质量 DoD

- [ ] 多租户跨读零次（同 03 文档）
- [ ] 单视频 1080p 起播 < 2s
- [ ] Tenant 注册到第一节视频可看 < 30 分钟（含转码等待）
- [ ] 计量数据可重算（重跑事件流得到一致结果）

### 8.3 业务 DoD

- [ ] 招募 5 位 friendly creator 完成首月使用
- [ ] 至少 3 位创作者上传 ≥ 1 门课程并产生学员观看
- [ ] 至少 1 位创作者触发"准备升级付费"的咨询

## 零、v0.1 → v0.2 重大决策

### ADR-101 微信集成路线（C 路线确定）

**决策时间：** 2026-05-16

**背景：** v0.1 原方案要求创作者持有「认证服务号 + 小程序 + 支付商户号」三件套，前置审批 6-12 周，单个创作者年合规成本高，违背「让创作者像运营 SaaS 一样运营自己的成长社群」的轻量化定位。

**决策：** 选 **C 路线（H5 + PWA + 公众号订阅消息 + 第三方付费）**。

**核心变更：**

| 维度 | v0.1 | v0.2 (C) |
|---|---|---|
| 学员端 | 微信小程序 | H5 + PWA |
| 付费 | 自营微信支付 | 寄生第三方（小报童/星球/有赞） |
| 公众号 | 认证服务号（必须） | 未认证订阅号（可选） |
| 服务商授权 | 必须 | 不需要 |
| 创作者前置审批 | 6-12 周 | 0-1 周 |
| 同舟 MVP 工时 | 12 周 | 8 周 |
| 学员主标识 | unionid（小程序握手） | 手机号（短信验证） |
| 数据完整性 | 强（支付回调） | 中（第三方对账） |

**加重路径预留：** V1 加微信支付，V2 加小程序，V2 加企业微信。详见 04 §10。

**取舍代价：** 失去 unionid 自动握手；牺牲一部分"轻应用"质感；学员入站环节多一步手机号验证；月推送通道少一条（无小程序订阅消息，但仍有公众号订阅消息）。

**触发推翻条件：** 灰度首营复盘中，若学员留存 < 30% 且明确归因于 H5 体验，则启动 V2 小程序路径。

### ADR-102 V0 加速起步路线（V0 → V0.5 → V1 → V2）

**决策时间：** 2026-05-16

**背景：** 8 周 MVP 仍包含完整训练营闭环（视频 + 作业 + 签到 + 群运营 + AI），存在两个问题：
1. **冷启动门槛高**：每位创作者首次使用要先有学员、有训练营计划、有微信资质 → 离付费意愿太远。
2. **未验证最小付费意愿**：作业 / 签到 / AI 是"高价值但非必需"的功能，应该让付费创作者按需"加重"。

**决策：** 把 8 周 MVP 拆为 4 阶段渐进交付：

| 阶段 | 工时 | 核心范围 | 商业目的 |
|---|---|---|---|
| **V0** | 4 周 | 轻量课程交付 + Freemium + 按量计费 | 拿到第一批付费创作者；验证「比小鹅通更轻」的定位 |
| V0.5 | +2 周 | 微信轻增强（公众号 / OAuth2 / 订阅消息 / PWA / 第三方付费 Webhook） | 提升留存触达 |
| V1 | +4 周 | 成长系统（作业 / 签到 / 班级 / Stage / AI 反馈） | 加重 ARPU，覆盖训练营场景 |
| V2 | +4 周 | 群运营 + AI 群日报 | 实现完整愿景 |

**核心约束：**
- 架构层完整保留 Track / Stage / Session / Cohort / Member 等模型
- V0 不创建 Cohort 实例（用 default rolling cohort 占位）
- V0 隐藏 Stage 层 UI，但 schema 保留
- V0 学员端从 4 Tab 砍为 3 个极简页面
- 多维度计量（学员 / 课程 / 存储 / 播放分钟）+ Freemium 字段是 V0 必做
- **学员观看不被中断**是商业承诺红线（即使创作者超额）

**取舍代价：**
- 与原始项目指令「禁止课程中心思维 / 必须成长路径」存在张力 → 通过"加速起步"框架化解：V0 = 入口，V1 = 实现长期定位
- 计费单价未定，可能影响商业模型早期判断 → 内测 1 月后定格
- V0 不接微信任何 API → 触达通道少（仅邮件 + 站内）

**推翻条件：** 若 V0 灰度 1 个月内 < 3 位创作者主动咨询升级，需复盘是否定位错误或定价错误。

# 同舟 · 数据模型与多租户隔离（v0.2）

> 数据库：PostgreSQL 16+
> ORM 建议：Prisma 或 Drizzle（TypeScript-first）
> **v0.2 变更（配合 V0 加速起步）**：
> - 新增 §3.5 计量与 Freemium 表（usage_events / usage_meters / tenant_quotas / usage_invoices）
> - `enrollments.cohort_id` 允许 NULL（V0 仅有 default rolling cohort，可省略实例）
> - 调整 `cohorts` 增加 `kind` 字段：`rolling`（V0 默认）/ `fixed_term`（V1 期次班级）
> - Members 表 `phone` 改为推荐而非强制（V0 学员匿名也能看）

## 1. 多租户策略选型

### 1.1 三种主流策略对比

| 策略 | 隔离强度 | 成本 | 复杂度 | 选用建议 |
|---|---|---|---|---|
| 每 Tenant 独立 DB | ★★★★★ | 高 | 高 | 大客户私有化时使用 |
| 共享 DB / 独立 Schema | ★★★★ | 中 | 中 | 中型 SaaS 常见 |
| **共享 DB / 共享 Schema + tenant_id + RLS** | ★★★★ | 低 | 低 | ✅ **MVP 选用** |

### 1.2 选择：共享 Schema + tenant_id + Row Level Security

**关键约束（红线，写入 lint 与 CI）：**

1. 业务表第一列必须是 `tenant_id UUID NOT NULL`
2. 所有索引都必须以 `tenant_id` 为前缀（即 `(tenant_id, ...)`）
3. 所有查询必须在事务开始时执行 `SET LOCAL app.tenant_id = '<uuid>'`
4. 表必须开启 RLS：`ALTER TABLE ... ENABLE ROW LEVEL SECURITY`
5. 默认策略：`USING (tenant_id = current_setting('app.tenant_id')::uuid)`

> 这一层是兜底保险。即便业务代码忘记加 tenant_id，数据库也会拒绝。

### 1.3 平台级表（不带 tenant_id）

- `platform_users`（平台运营人员）
- `tenants`（租户元数据本身）
- `audit_logs_platform`
- `subscription_plans`

### 3.5 计量与 Freemium（V0 新增）

> 详细模型见 07 文档 §4。这里给最终 schema。

#### `usage_events`（事件流，原子级）

```sql
CREATE TABLE usage_events (
  tenant_id   uuid NOT NULL REFERENCES tenants(id),
  id          uuid PRIMARY KEY,
  metric_key  text NOT NULL,            -- members.active_count / courses.count / storage.bytes / playback.minutes
  delta       numeric(20,4) NOT NULL,
  ref_kind    text NULL,                -- member / video / course / snapshot
  ref_id      uuid NULL,
  occurred_at timestamptz NOT NULL DEFAULT now(),
  meta        jsonb NOT NULL DEFAULT '{}',
  UNIQUE (tenant_id, id)
);

CREATE INDEX usage_events_tm_idx
  ON usage_events (tenant_id, metric_key, occurred_at DESC);

ALTER TABLE usage_events ENABLE ROW LEVEL SECURITY;
ALTER TABLE usage_events FORCE ROW LEVEL SECURITY;
CREATE POLICY tenant_isolation ON usage_events
  USING (tenant_id = current_setting('app.tenant_id', true)::uuid)
  WITH CHECK (tenant_id = current_setting('app.tenant_id', true)::uuid);
```

#### `usage_meters`（月聚合，幂等）

```sql
CREATE TABLE usage_meters (
  tenant_id     uuid NOT NULL REFERENCES tenants(id),
  id            uuid PRIMARY KEY,
  metric_key    text NOT NULL,
  period        text NOT NULL,              -- 'YYYY-MM'
  current_value numeric(20,4) NOT NULL DEFAULT 0,
  free_quota    numeric(20,4) NOT NULL DEFAULT 0,
  billable_value numeric(20,4)
    GENERATED ALWAYS AS (GREATEST(0, current_value - free_quota)) STORED,
  unit_price    numeric(20,6) NULL,         -- 月度生成账单时快照
  updated_at    timestamptz NOT NULL DEFAULT now(),
  UNIQUE (tenant_id, metric_key, period)
);
-- RLS 同上
```

#### `tenant_quotas`（Freemium 阈值与计价，字段预留）

```sql
CREATE TABLE tenant_quotas (
  tenant_id    uuid PRIMARY KEY REFERENCES tenants(id),
  plan         text NOT NULL DEFAULT 'free',  -- free / pro / custom
  effective_at timestamptz NOT NULL DEFAULT now(),
  quotas       jsonb NOT NULL DEFAULT '{}'    -- 见 07 §4.3
);
```

**`quotas` JSON 结构（V0 预留）：**

```json
{
  "members.active_count":  { "free": null, "unit_price": null, "currency": "CNY" },
  "courses.count":         { "free": null, "unit_price": null, "currency": "CNY" },
  "storage.bytes":         { "free": null, "unit_price": null, "currency": "CNY", "unit": "GB·月" },
  "playback.minutes":      { "free": null, "unit_price": null, "currency": "CNY", "unit": "千分钟" }
}
```

> 注意：`tenant_quotas` 是平台级配置之一，**不带 RLS**（同舟运营写入，每个 Tenant 通过 join 读自己一行即可），但仍需服务端在读路径校验当前 tenant 上下文。

#### `usage_invoices`（账单）

```sql
CREATE TABLE usage_invoices (
  tenant_id    uuid NOT NULL REFERENCES tenants(id),
  id           uuid PRIMARY KEY,
  period       text NOT NULL,
  status       text NOT NULL,                -- draft / issued / paid / void
  total_amount numeric(20,4) NOT NULL DEFAULT 0,
  currency     text NOT NULL DEFAULT 'CNY',
  lines        jsonb NOT NULL DEFAULT '[]',  -- [{metric_key, billable, unit_price, subtotal}, ...]
  issued_at    timestamptz NULL,
  paid_at      timestamptz NULL,
  UNIQUE (tenant_id, period)
);
-- RLS 同上
```

#### 计量采集来源（实现说明）

| metric_key | 采集时机 | 实现位置 |
|---|---|---|
| `members.active_count` | 学员任意访问 H5（含视频播放） | API 中间件，按 `(tenant, member, day)` 去重，每日首次访问写一条 `delta=1` |
| `courses.count` | 课程发布 / 下架 | Track 状态机回调 |
| `storage.bytes` | 每日凌晨任务 | OSS / VOD bucket 用量同步任务 |
| `playback.minutes` | 播放器 30 秒心跳 | `POST /api/v1/m/playback/heartbeat` |

#### 月度对账任务（Cron）

```
每日 03:00  → 重算昨日的 usage_meters.current_value
每月 1 日 04:00 → 给上月各 metric 生成 usage_invoice（draft 状态）
                  → 邮件通知创作者「上月用量与预估费用」
每月 5 日 → invoice issued（正式开账，状态变 issued）
```

> 对账幂等性：`usage_meters.current_value` 必须可由 `usage_events` 完整重算得到。CI 中包含重算一致性测试。

## 4. RLS 策略示例（PostgreSQL）

```sql
-- 1. 业务表创建
CREATE TABLE submissions (
    tenant_id  uuid NOT NULL REFERENCES tenants(id),
    id         uuid PRIMARY KEY,
    session_id uuid NOT NULL,
    cohort_id  uuid NOT NULL,
    member_id  uuid NOT NULL,
    status     text NOT NULL,
    content    jsonb NOT NULL,
    submitted_at timestamptz NOT NULL DEFAULT now(),
    -- ...
    UNIQUE (tenant_id, id)
);

CREATE INDEX submissions_session_idx
  ON submissions (tenant_id, session_id, submitted_at DESC);

-- 2. 启用 RLS
ALTER TABLE submissions ENABLE ROW LEVEL SECURITY;
ALTER TABLE submissions FORCE ROW LEVEL SECURITY;

-- 3. 策略：当前会话变量 = 行的 tenant_id
CREATE POLICY tenant_isolation ON submissions
  USING (tenant_id = current_setting('app.tenant_id', true)::uuid)
  WITH CHECK (tenant_id = current_setting('app.tenant_id', true)::uuid);

-- 4. 应用层每次开事务：
-- BEGIN;
-- SET LOCAL app.tenant_id = '...';
-- ...业务 SQL...
-- COMMIT;
```

**双保险机制（建议）：**

- 应用层（ORM 中间件）：所有查询自动注入 `WHERE tenant_id = $current_tenant`
- 数据库层（RLS）：兜底
- 测试层：CI 中运行"反向测试"，验证未设置 tenant_id 时所有读写都被拒绝

## 6. 数据生命周期与软删除

- 业务表统一加 `deleted_at timestamptz NULLABLE`
- RLS 策略默认排除 `deleted_at IS NOT NULL`
- 平台级别的删除策略：
  - Tenant 暂停 → 数据保留
  - Tenant 注销 → 30 天软删 → 90 天硬删（合规：导出权）

## 7. 与 Supabase / Prisma 的衔接备忘

- 即便使用自建 Postgres，也可以参考 Supabase 的 RLS 范式
- 若用 Prisma：通过 `$transaction + $executeRawUnsafe('SET LOCAL ...')`
- 若用 Drizzle：通过 middleware 在每个连接 checkout 时 `SET app.tenant_id`

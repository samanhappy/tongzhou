# 同舟（Tongzhou）

> 面向创作者的微信成长社群 SaaS 操作系统
> 让每一个创作者，都能拥有自己的「成长型 SaaS 产品」

---

## 0. 当前阶段：V0「轻量化课程交付 SaaS」（加速起步路径）

**长期愿景：** 微信成长社群 SaaS 操作系统
**V0 切入：** 一个比小鹅通更轻、更便宜的视频课程交付 SaaS

V0 范围（4 周交付）：
- 课程上传 + 视频交付（OSS + VOD + 防盗链 + 水印）
- 学员 H5 落地页（极简：分享链接 → 课程目录 → 播放）
- 多维度用量计量（学员数 / 课程数 / 存储 / 播放分钟）
- Freemium quota + 超额降级
- **创作者零微信资质要求**（不需要公众号、不需要支付商户号）

V0 之后渐进：V0.5（微信轻增强）→ V1（成长系统：作业/签到/班级/AI）→ V2（群运营 + AI 群日报）。

详见 [07-V0范围与计量模型.md](./07-V0范围与计量模型.md) 与 [05-MVP与路线图.md](./05-MVP与路线图.md)。

## 三、核心定位（强约束摘要）

- **不是**：知识付费课程平台、LMS、IM、电商、CRM
- **是**：让创作者像运营 SaaS 一样运营自己的成长社群与训练营
- **三层结构**：SaaS 平台层 + 成长社群交付层 + 微信生态增强层
- **关键原则**：SaaS 优先 + WeChat-first + 成长优先（非内容优先）

---

## 四、技术栈选型（一句话）

> **Next.js**（创作者后台 + 学员 H5 + BFF） + **Node**（Fastify / NestJS）后端 + **PostgreSQL**（共享 Schema + tenant_id + RLS） + **Redis** + **阿里云 VOD / OSS**

## 四之一、关键路线决策（v0.3）

> **V0（4 周）= 轻量化课程交付 + Freemium + 按量计费**
> 创作者零微信资质，学员匿名可看；微信集成全部推到 V0.5。
>
> **V0.5（+2 周）= 微信轻增强**
> 公众号订阅号 + OAuth2 + 订阅消息 + PWA + 第三方付费 Webhook。
>
> **V1/V2 = 成长系统补全 + 群运营**
> 详见 [05-MVP与路线图.md](./05-MVP与路线图.md) §2 与 [99 §零 ADR-102](./99-自检与开放问题.md)。

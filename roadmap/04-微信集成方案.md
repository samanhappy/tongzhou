# 同舟 · 微信生态集成方案（v0.3）

> **v0.2 → v0.3 关键调整**：配合 V0 加速起步，把 v0.2（C 路线 · H5+订阅消息+第三方付费）拆为分阶段实施：
> - **V0（4 周）：几乎不接微信。** 仅极简 H5 + 邮件 + 站内通知。学员匿名访问，可选手机号绑定。
> - **V0.5（+2 周）：微信轻增强。** 加公众号订阅号 + OAuth2 + 订阅消息 + PWA + 第三方付费 Webhook。
> - **V1+：** 视情况上微信支付 / 自营小程序（加重路径）。
>
> 设计目标：**V0 阶段创作者零微信资质，连订阅号都不强求。**

---

## 0. V0 阶段（最小集合）

> V0 不接任何微信 API。整体微信触点仅 3 个：
>
> 1. 创作者把 **同舟分享链接** 发到自己的微信群 / 朋友圈
> 2. 学员在微信内浏览器打开链接 → 进入同舟 H5 → 看视频
> 3. 学员可选 **绑定手机号** 记录跨设备进度（短信验证）

**V0 阶段唯一与微信"相关"的事：**

- H5 页面针对**微信内置浏览器**做兼容（视频自动播放策略、长按弹菜单等）
- 分享链接生成时，提供"复制链接 + 复制文案 + 二维码图"三件套，方便创作者粘贴到微信群

**V0 阶段同舟主动避免：**

- 不申请公众号（哪怕是订阅号）
- 不申请支付商户号
- 不接 OAuth2、不接订阅消息
- 不做 PWA（V0.5 才做）

> V0 红线：**绝不让创作者卡在微信资质上**。

## 1. V0.5 路线：核心理念

> **同舟负责交付与运营增强；付费交给创作者已有的工具；微信只用最轻的一层。**

以下内容描述 V0.5 阶段才接入的微信能力。V0 不实施。

```
   创作者                同舟做的事                  外部合作
   ───────              ──────────                  ─────────
   收钱  ───────────►   ❌ 不做             ─►   知识星球 / 小报童 /
                                                  有赞 / 飞书付费等
   学员名单收集  ───►   ✅ 自动同步 + 手动导入

   视频/作业/签到/AI  ─►  ✅ 核心能力（H5 + PWA）

   推送提醒  ────────►   ✅ 公众号订阅消息 (轻量)
                        ✅ 微信群手动 @ 文案生成

   微信群运营增强  ───►   ✅ 半自助结构化（同 v0.1）
```

## 2. 创作者需要持有的资质（三阶段渐进）

| 资质 | V0（4 周） | V0.5（+2 周） | V1+ |
|---|---|---|---|
| 微信认证服务号 | ❌ 不需要 | ⚪ 可选 | ⚪ 可选 |
| 未认证订阅号 | ❌ 不需要 | ✅ 推荐 | ✅ 推荐 |
| 微信小程序 | ❌ | ❌ | ⚪ V2 加重路径 |
| 微信支付商户号 | ❌ | ❌ | ⚪ V2+ 加重路径 |
| 第三方付费工具账号 | ⚪ 自己用，同舟不接 | ✅ Webhook 接入 | ✅ |
| 群活码 / 群二维码 | ⚪ 创作者自己发 | ⚪ 自己发 | ✅ 半自助绑定 |

**V0 结论：** **最小可用配置 = 一个邮箱**。
**V0.5 结论：** 一个未认证订阅号（5 分钟开，0 元）+ 一个第三方付费工具账号。

## 3. 学员端形态：H5 + PWA

### 3.1 为什么 H5 而不是小程序

- ⏱ **免审核**：今天写，今天上
- 🎨 **品牌独立**：创作者域名 `creator.com/m` 即可，无需小程序前缀
- 🔁 **迭代快**：发版无需小程序审核 7 天
- 🌍 **覆盖外部场景**：可在飞书、知乎、播客 shownote 等任意载体跳转
- 🚫 **代价**：失去微信「轻应用」感、首次进入需手动添加到桌面 / 收藏

### 3.2 用 PWA 弥补"轻应用"体验

- 学员首次进入引导"添加到主屏"（iOS Safari、Chrome / 微信浏览器有限支持）
- Service Worker 缓存 → 二次打开几乎瞬开
- 离线访问已下载的视频海报 / 文字资料
- 推送：iOS 16.4+ 支持 Web Push，Android 早已支持
- 微信内浏览器限制：禁用 Web Push、禁用 PWA install banner → 仍以**公众号订阅消息** 兜底

### 3.3 学员端域名策略

- 默认：`m.tongzhou.app/t/<tenant-slug>/...`
- 付费用户可自定义：`xxx.com/m/...`（CNAME + Let's Encrypt 自动证书）
- 微信内分享卡片：H5 链接通过公众号"自动回复 + 跳转"实现

## 4. 付费链路：寄生 + 同步

### 4.1 三种付费模式（创作者按现状选）

| 模式 | 创作者用什么 | 同舟怎么拿到学员 | 适合 |
|---|---|---|---|
| **A. 知识星球付费入星** | 已有星球 | 学员主动报名 → 创作者批量导出 → 同舟导入 | 已有星球的 IP |
| **B. 小报童 / 有赞付费表单** | 第三方付费表单 | Webhook 接入 / CSV 周期同步 | 单期训练营 |
| **C. 私下转账 / 收款码** | 自家收款 | 创作者手动添加学员（粘贴名单） | 小规模 |
| **D. 微信群红包接龙 / 群裂变** | 群运营 | 创作者手动操作 | 极小规模 |

### 4.2 学员加入同舟的统一入口

```
[创作者] 在 同舟后台「学员」页 → 添加学员
   ┌─ 单个添加：填手机号 + 昵称 → 系统生成入营链接
   ├─ 批量导入：上传 CSV（手机号, 昵称[, 微信昵称]）
   ├─ 自动同步：Webhook 接第三方付费工具（V1 接入有赞/小报童）
   └─ 二维码报名：生成「我已付款，加入学习」H5 表单，学员填手机号自助加入

[学员] 收到入营链接（短链 / 二维码 / 群内分享）
        ▼
        点开 → H5 落地页
        ▼
        授权微信（如在微信内）/ 输入手机号验证（如在外部浏览器）
        ▼
        进入「今日」首页
```

### 4.3 学员身份解析

```
                          ┌─────────────────┐
                          │  member         │
                          │  (Tenant 内身份) │
                          └────────┬────────┘
                                   │
              ┌────────────────────┼────────────────────┐
              ▼                    ▼                    ▼
   ┌──────────────────┐  ┌──────────────────┐  ┌──────────────┐
   │ phone (短信验证) │  │ wechat openid    │  │ third-party  │
   │ 主要标识         │  │ (公众号关注后)   │  │ ref (星球/有赞 │
   │                  │  │                  │  │  user_id)    │
   └──────────────────┘  └──────────────────┘  └──────────────┘
```

**关键：** 没有小程序，就没有 `wx.login` 的 unionid 自动握手。同舟选 **手机号作为 Member 主标识**，公众号 openid 是绑定附加项（用于推送）。

## 5. 推送通道（按可用性优先级）

### 5.1 通道矩阵

| 通道 | 触达率 | 创作者成本 | 同舟实现成本 | 用途 |
|---|---|---|---|---|
| 公众号订阅消息 | 高（需授权） | 未认证订阅号即可 | 中 | 主：每日 Session 提醒、作业反馈 |
| 微信群手动 @ | 高 | 0 | 极低（生成文案） | 主：每日打卡 / 未签到名单 |
| 短信 | 极高 | 几分/条 | 低 | 备：关键操作 / 退款 |
| 站内提醒（H5 红点） | 中 | 0 | 极低 | 默认 |
| 邮件 | 低 | 0 | 极低 | 创作者侧通知（学员几乎不开） |
| iOS / Android Web Push | 中 | 0 | 中 | PWA 用户的附加通道 |

### 5.2 公众号订阅消息策略（轻量版）

**未认证订阅号也能用一次性订阅消息**（2020 年后的能力）。规则与服务号订阅消息基本一致：

- 学员每次主动点击"接收提醒"按钮 → 弹出微信原生授权 → 可发 1 次
- 推送的链接可以直接打开 H5（同一公众号关联的网页）
- 不再需要服务号 → 没有年审、没有 ¥300

**实现：**

```
学员 H5 首页:
   ┌──────────────────────────────┐
   │ 接收今日训练提醒              │
   │ [开启提醒] (按钮)             │
   └──────────────────────────────┘
              ▼ 点击
   微信原生授权（一次最多 3 个模板）
              ▼ 确认
   后端记录 wechat_subscribe_tokens.remaining += 1
              ▼ 之后某天
   到点触发 → 后端扣 1 次 → 调 sendSubscribeMessage
```

> 维护好 `remaining`：每次推送扣 1，到 0 时不再发，下次学员主动操作时再补充。

### 5.3 微信群通道（核心增量保留）

- 同舟生成「今日开营文案」「未签到 @ 名单」「精华转述」
- 创作者一键复制 → 粘贴到群里发
- **不自动发**：100% 合规

## 6. 微信开放平台只用最薄的一层

### 6.1 必接

- **网页授权登录 OAuth2**（公众号网页授权）：在微信内打开 H5 时拿到 openid
  - 接 `snsapi_base` 静默授权拿 openid
  - 接 `snsapi_userinfo` 需用户确认，拿昵称头像

### 6.2 选接

- **JS-SDK**：分享卡片、扫一扫、获取地理位置等增强体验
- **订阅消息 API**：见上节

### 6.3 完全不接

- 微信支付（→ 第三方）
- 小程序所有 API
- 服务商授权（→ 创作者直连自己的订阅号即可）

## 9. 风险与缓解

| # | 风险 | 缓解 |
|---|---|---|
| W1 | 微信内 H5 体验"不像应用" | PWA + 首屏引导"添加到主屏" + 公众号底部菜单一键直达 |
| W2 | 微信浏览器禁 Web Push | 退化到公众号订阅消息 |
| W3 | 第三方付费工具断 Webhook | 提供 CSV 周期上传兜底 |
| W4 | 学员无 openid 推送不到 | 强引导学员关注公众号；短信兜底 |
| W5 | 微信对 H5 的限制收紧 | 同舟保持"自有域名 + 多触达通道"，最坏情况切换到飞书 / 小报童 H5 |
| W6 | 反爬 / 滥用 | H5 链接带 tenant + member 签名 + 短期失效 |

## 10. 升级路径（C → B → A）

C 路线不是终点，是 MVP 阶段的起点。验证后可分阶段加重：

```
MVP（C）          V1                 V2                  V3
─────────         ─────────          ─────────           ─────────
H5 + PWA          + 微信支付         + 自营小程序        + 企业微信
公众号订阅消息    (服务商分账)        (品牌独立)         (群消息自动化)
第三方付费同步    (走向 toC)         (走向规模化)        (走向自动化)
```

**升级触发器：**

- 客户提出"我希望学员看到我的品牌而不是同舟"→ 加自定义域名（V1 内）
- 客户提出"我不想用第三方收钱，要直接微信支付"→ 加服务商分账（V1）
- 客户单期超过 500 学员、流量大 → 加小程序（V2）
- 客户提出"我希望群消息自动结构化"→ 加企微（V2）
